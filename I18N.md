# Mega X · i18n Plan / 国际化方案

> Bilingual (English + Chinese) launch for the whole Mega X site — marketing
> static pages **and** the Phyntom X8 Console SPA — with a deliberate runway
> for Arabic (RTL) as the third locale.
>
> 整个 Mega X 站（营销静态页 + Phyntom X8 Console SPA）的双语（英文 + 中文）方案，
> 并预留阿拉伯语（RTL 从右到左）作为第三语言的扩展路径。

## 1. Goals & Constraints / 目标与约束

**EN**
- **Default language is English.** Root URL `/` shows English. Chinese visitors must opt in.
- Add Chinese as the second locale. The user-facing brand stays "Phyntom X8" / "Mega X" — only descriptive text translates.
- Prepare for Arabic (RTL) without writing Arabic strings now: every CSS rule and layout decision must work in both LTR and RTL from day one.
- Single domain `mega-x.ai` (no subdomain per locale).
- SEO must work: each language has its own crawlable URL with proper `<html lang>` and `hreflang` links.

**中**
- **默认语言 = 英文**。根路径 `/` 渲染英文。中文访客需主动切换。
- 中文作为第二语言。品牌名（Phyntom X8 / Mega X）保留，只有描述性文本翻译。
- 现在不写阿拉伯文，但所有 CSS 和布局必须**从零开始**兼容 LTR 和 RTL。
- 仅一个域名 `mega-x.ai`（不按语言分子域名）。
- SEO 必须工作：每种语言要有自己可被爬虫索引的 URL，正确的 `<html lang>` 与 `hreflang` 标签。

## 2. URL & Routing Strategy / 路由策略

**Path-prefix model**（路径前缀模型）— industry-standard, SEO-friendly, no DNS work.

| Locale | Marketing Home | Marketing Detail | Console |
|---|---|---|---|
| English (default) | `/` | `/phyntom-x8.html` | `/console/`, `/console/business/` |
| Chinese | `/zh/` | `/zh/phyntom-x8.html` | `/console/?lang=zh` *(see below)* |
| Arabic (future) | `/ar/` | `/ar/phyntom-x8.html` | `/console/?lang=ar` |

**Marketing pages (MPA)** use full path prefix `/zh/...` — separate HTML files per locale (best for SEO, each page is a real file).

**Console SPA** stays at `/console/*` (no `/zh/console/`) because:
- It's a single-page React app with all translations bundled
- Re-routing the entire SPA per locale doubles asset emission for no SEO gain (SPA content isn't crawled the same way as static HTML)
- Locale switched via `localStorage` + optional `?lang=` URL param

**中文版**：营销页用路径前缀 `/zh/...`（每语言一份静态 HTML，SEO 友好）。Console SPA 保持 `/console/*`，语言通过 `localStorage` 切换；可选 `?lang=zh` URL 参数用于分享/书签。

## 3. Technical Architecture / 技术架构

### 3.1 Marketing MPA (vanilla HTML + Vite partials plugin)

**EN**
- Translations live in `i18n/{locale}.json` (centralized) — keys flat or nested per page
- Extend `tools/vite-plugin-partials.ts` with locale awareness: for each entry HTML, render once per locale, emit to `dist/<locale>/<file>.html` (English emits to `dist/<file>.html` at root)
- New placeholder syntax inside partial & body templates: `{{t:home.hero.title}}` → looked up in active locale JSON
- `partials/pages.json` gains per-locale overrides for canonical / og:url:
  ```json
  "phyntom-x8.html": {
    "title": { "en": "...", "zh": "..." },
    "description": { "en": "...", "zh": "..." }
  }
  ```
- Build output:
  ```
  dist/
  ├── index.html              ← English (default at /)
  ├── about.html
  ├── phyntom-x8.html
  ├── zh/
  │   ├── index.html
  │   ├── about.html
  │   └── phyntom-x8.html
  └── ar/                     ← future
      └── (mirror)
  ```

**中**
- 翻译文本集中在 `i18n/{locale}.json`
- 扩展 [tools/vite-plugin-partials.ts](tools/vite-plugin-partials.ts) 支持 locale：每个 HTML 入口为每种语言渲染一次，输出到 `dist/<locale>/<file>.html`（英文为默认，直接到 `dist/<file>.html`）
- 在 partial 和 body 模板里新增占位符 `{{t:home.hero.title}}` —— 从当前 locale 的 JSON 查找替换
- `partials/pages.json` 的 title/description/og 字段改成 per-locale 对象
- 构建产物如上结构

### 3.2 Console SPA (React + Vite)

**EN**
- Use **`react-i18next`** — the default React i18n library (well-maintained, MIT, RTL-ready)
- Translations under `console/src/i18n/{locale}.json`
- `<I18nextProvider>` wraps the React tree in `main.tsx`
- All visible text wrapped in `t('key')` from `useTranslation()` hook
- Locale persisted in `localStorage` (key `phyntom.locale`); read on app boot
- Optional `?lang=zh` URL param overrides storage on first visit (for shareable links)
- Language switcher in `ConsoleShell` header — a `<select>` or dropdown
- Mock data in `lib/mocks.ts` can stay English (data is mock anyway); add localized variants in S1+ when real backend lands

**中**
- 用 `react-i18next` —— React 生态默认选择
- 翻译放 `console/src/i18n/{locale}.json`
- `main.tsx` 顶层包 `<I18nextProvider>`
- 组件里所有可见文本用 `useTranslation()` 的 `t('key')` 包起来
- 当前 locale 存 `localStorage`（key 名 `phyntom.locale`），启动时读出
- 可选 URL 参数 `?lang=zh` 覆盖 storage（便于分享链接）
- `ConsoleShell` header 加语言切换器（下拉框）
- mock 数据先英文为主（反正是 mock），等 S1 真后端再做多语种数据

### 3.3 RTL Preparation / RTL 准备

**EN — invariant CSS rules from day one**
- ❌ **Never use** `margin-left`, `padding-right`, `left:`, `right:`, `text-align: left`
- ✅ **Always use logical properties**: `margin-inline-start`, `padding-inline-end`, `inset-inline-start`, `text-align: start`
- ❌ Never `flex-direction: row` with assumed L→R semantics; use `flex-direction: row` knowing it auto-mirrors under `dir="rtl"`
- ✅ Icons that direct meaning (← → ➜) need RTL variants OR use semantic SVGs with `transform: scaleX(-1)` under `[dir="rtl"]`
- ✅ `<html lang="..." dir="...">` is the single switching point — Tailwind reads it via `[dir="rtl"]:` modifier

**Tailwind config additions**:
```ts
// tailwind.config.ts
plugins: [
  // ...existing
  // First-class RTL utilities: `ms-4` (margin-inline-start), `pe-2`, `text-start`
  // (these are built into Tailwind 3.3+)
]
```

**中**
- ❌ **永远不用** `margin-left` / `padding-right` / `left:` / `right:` / `text-align: left`
- ✅ **始终用逻辑属性**：`margin-inline-start` / `padding-inline-end` / `inset-inline-start` / `text-align: start`
- ✅ Tailwind 3.3+ 已内建 logical 工具类：`ms-4`、`pe-2`、`text-start`
- ✅ 单一切换点：`<html lang="ar" dir="rtl">` —— 所有 CSS 通过 `[dir="rtl"]:` 选择器自动适配

### 3.4 Font Strategy / 字体策略

| Locale | Display Font | Body Font |
|---|---|---|
| English | `Instrument Serif` | `Barlow` |
| Chinese | system fallback (`PingFang SC`, `Hiragino Sans GB`, `Microsoft YaHei`) + `Barlow` Latin glyphs | same |
| Arabic | `Vazirmatn` / `Noto Naskh Arabic` (Google Fonts) | same |

**EN** — Define language-aware font stacks in `tailwind.config.ts` via CSS variables:
```css
:root { --font-display: 'Instrument Serif', serif; }
[lang="zh"] { --font-display: 'Instrument Serif', 'PingFang SC', 'Microsoft YaHei', sans-serif; }
[lang="ar"] { --font-display: 'Vazirmatn', 'Instrument Serif', serif; }
```

**中** — 通过 CSS 变量绑定语言到字体栈。`<html lang="zh">` 自动切换。

## 4. Translation File Layout / 翻译文件布局

```
frontend/mega-x/
├── i18n/                                  ← marketing MPA translations
│   ├── en.json                            ← English (default)
│   ├── zh.json                            ← Chinese
│   └── ar.json                            ← Arabic (future, stub for now)
│
└── console/src/i18n/                      ← Console SPA translations
    ├── en.json
    ├── zh.json
    └── ar.json
```

**Schema** — flat keys with dot-namespacing:

```json
{
  "common.nav.home": "Home",
  "common.nav.console": "Console",
  "common.footer.copyright": "© 2026 Mega X Holding Ltd.",
  "home.hero.headline": "MEGA X",
  "home.hero.epigraph": "Join Us on Our Journey.",
  "phyntom-x8.hero.eyebrow": "Phyntom X8 · Certified AI Department",
  "phyntom-x8.cta.open-console": "Open Console →"
}
```

Keys are **deterministic from page + section + element**; no creative naming. This keeps `en.json` ↔ `zh.json` ↔ `ar.json` 1:1 mirrored so missing keys are detectable.

**中文版** —— 平铺键 + 点命名空间。三种语言文件键集必须完全 1:1，缺一项要被脚本检出。

## 5. Language Switcher UX / 语言切换器

**Header dropdown** (top-right of nav, both marketing and Console):

```
┌─────────────────────┐
│  EN ▼               │
│  ─────              │
│  English   ✓        │
│  中文                │
│  العربية (soon)      │
└─────────────────────┘
```

**EN — Behavior**
- Click switches to the equivalent URL in target locale (`/about.html` → `/zh/about.html`)
- For Console, updates `localStorage` and triggers a re-render (no URL change for SPA pages)
- Initial visit: detect `navigator.language`. If matches a supported locale, **suggest** via a dismissable banner — never auto-redirect (bad SEO + UX)
- Persisted choice in `localStorage` (marketing) and Console — same key `phyntom.locale`

**中**
- 点击跳到对应语言的等价 URL（营销页路径换前缀；Console 写 localStorage 并立即重渲）
- 首次访问按 `navigator.language` **建议**（不自动跳转，避免坏 SEO 与 UX）
- 用户选择持久化到 `localStorage`

## 6. SEO Tags / SEO 标签

Every page (per locale) must emit:

```html
<!-- in <head> -->
<html lang="en">                                                  <!-- or zh / ar -->
<link rel="canonical" href="https://mega-x.ai/phyntom-x8.html">
<link rel="alternate" hreflang="en" href="https://mega-x.ai/phyntom-x8.html">
<link rel="alternate" hreflang="zh" href="https://mega-x.ai/zh/phyntom-x8.html">
<link rel="alternate" hreflang="ar" href="https://mega-x.ai/ar/phyntom-x8.html"> <!-- future -->
<link rel="alternate" hreflang="x-default" href="https://mega-x.ai/phyntom-x8.html">
```

`hreflang` tells Google which version to show users from which region. `x-default` is the fallback (English).

These are emitted by the extended `vite-plugin-partials.ts` automatically from a single source — never hand-written.

## 7. Implementation Phases / 实施阶段

### Phase 0 — Inventory & Extract (1 day)
- Walk every marketing HTML, list every user-visible string
- Walk every Console TSX, list every JSX text
- Output: a single CSV / spreadsheet with `key | english_source | chinese_translation`
- Send to translator (or use AI draft + human review)

### Phase 1 — Marketing MPA i18n Build (1-2 days)
- Extend `tools/vite-plugin-partials.ts` to read `i18n/{locale}.json` and substitute `{{t:key}}` tokens
- Modify `pages.json` schema to support per-locale title/description
- Source HTMLs get `{{t:home.hero.headline}}` markers replacing inline text
- Build emits `dist/` (English) + `dist/zh/` (Chinese)
- Add `<link rel="alternate" hreflang>` tags automatically
- Language switcher in `partials/nav.html`

### Phase 2 — Console SPA i18n (1-2 days)
- Add `react-i18next` + `i18next` deps
- Wrap `main.tsx` in `<I18nextProvider>`
- Create `console/src/i18n/{en,zh}.json`
- Replace all hardcoded JSX text with `t('key')` (mechanical refactor; ~50-100 strings)
- Language switcher in `ConsoleShell` header
- Persist locale to `localStorage`; read on boot

### Phase 3 — Real Translations (depends on translator turnaround)
- Translator fills `zh.json` from extraction CSV
- QA pass: every page rendered in English + Chinese, no missing keys, no overflowing text

### Phase 4 — RTL Audit & Logical CSS Migration (1 day)
- Run a script that flags every `margin-left`, `padding-right`, `text-align: left` in `styles/**/*.css`
- Replace with `margin-inline-start` / `padding-inline-end` / `text-align: start`
- Test by manually setting `<html dir="rtl">` in DevTools — verify all layouts mirror correctly
- This is the "lock-in" point — after this, Arabic is just a JSON file

### Phase 5 (future) — Add Arabic
- Translator fills `ar.json`
- Add Arabic to language switcher dropdown
- Add `dir="rtl"` switching logic
- Add Arabic font (`Vazirmatn` via Google Fonts)
- Smoke-test all pages render correctly RTL

## 8. Decisions Already Locked / 已锁定决策

| Decision | Choice |
|---|---|
| Default language | English |
| Second language | Chinese |
| Future language | Arabic (RTL) |
| URL pattern | Path prefix (`/zh/`, `/ar/`) |
| Console URL | Stays `/console/*` (no per-locale path); locale via storage + `?lang=` |
| Marketing i18n engine | Extended `vite-plugin-partials.ts` (no new lib) |
| Console i18n engine | `react-i18next` |
| Translation files | `i18n/{locale}.json` (marketing) + `console/src/i18n/{locale}.json` (SPA) |
| Key naming | Flat dot-namespaced (`page.section.element`) |
| Auto-redirect on first visit | **No** (banner suggestion only) |
| Locale persistence | `localStorage["phyntom.locale"]` |
| RTL CSS | Logical properties (`-inline-start`, `-inline-end`) only |
| Font strategy | CSS-variable-driven per `[lang]` selector |

---

## 9. Out of Scope (for now) / 暂不处理

- ❌ **Auto-detect + auto-redirect** based on `Accept-Language` — bad UX, hurts SEO
- ❌ **Per-locale subdomains** (`zh.mega-x.ai`) — no DNS budget, path prefix is fine
- ❌ **Translating Console mock data** to Chinese — wait until S1 real backend
- ❌ **CMS-based translation workflow** (e.g. Crowdin / Lokalise) — JSON files in git is enough at this scale
- ❌ **Pluralization rules** (`ICU MessageFormat`) — `react-i18next` supports it but not needed for current copy
